# Frequently Asked Questions

## General

### What is Booster Syndrome?
Booster Syndrome is a comprehensive microservices platform built with multiple programming languages, demonstrating modern cloud-native architecture patterns.

### What languages are used?
- Rust (Auth, File Storage)
- Go (Gateway, Message Queue, Metrics)
- TypeScript/Node.js (User Service, Frontend)
- Python (Analytics)
- Ruby (Notifications)
- Elixir (Search, Chat)

### Is this production-ready?
This is a demonstration project showcasing various technologies and patterns. For production use, additional hardening, testing, and security measures would be needed.

## Installation

### What are the prerequisites?
- Docker & Docker Compose
- Rust 1.70+
- Go 1.21+
- Node.js 18+
- Python 3.11+
- Ruby 3.2+
- Elixir 1.15+

### How do I install it?
```bash
git clone https://github.com/ikaydreams-dev/Booster-Syndrome.git
cd Booster-Syndrome
./scripts/setup/install.sh
```

### Can I run individual services?
Yes, each service can run independently. See the service's README for specific instructions.

## Development

### How do I run tests?
```bash
make test
```

### How do I add a new service?
1. Create service directory
2. Implement service logic
3. Add Dockerfile
4. Update docker-compose.yml
5. Add to CI/CD pipeline

### How are services authenticated?
Services use JWT tokens for authentication, generated by the Auth service.

## Troubleshooting

### Service won't start
- Check logs: `docker-compose logs service-name`
- Verify environment variables
- Ensure dependencies are running

### Database connection errors
- Check DATABASE_URL environment variable
- Verify database is running
- Run migrations: `./scripts/migrations`

### API returns 401
- Check JWT token is valid
- Verify Authorization header
- Token may be expired

## Architecture

### Why microservices?
Demonstrates modern distributed system patterns, language polyglotism, and scalability.

### Why multiple languages?
To showcase how different languages can work together in a microservices architecture, each chosen for its strengths.

### How do services communicate?
- Synchronous: REST APIs
- Asynchronous: RabbitMQ message queue
- Real-time: WebSockets
